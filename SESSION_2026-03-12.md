## Key Learnings

### Executive Brief Validation is Strict and Effective
The validation pipeline catches multiple quality issues:
1. **Section title mismatches** - Rejects if LLM uses wrong titles ("Opening" instead of "Executive Summary")
2. **Insufficient numeric values** - Requires ≥3 per Key Finding insight, ≥15 total
3. **Fallback text usage** - Detects placeholder content and triggers retry
4. **CRITICAL/HIGH severity** - Enforces substantive insights when critical findings exist

This strictness occasionally causes retries (as seen tonight), but ensures high-quality output.

### Performance Trade-offs
- Current design prioritizes **quality over speed**
- LLM agents use low temperature (0.0-0.2) for consistency
- Structured JSON generation adds latency but provides reliability
- Validation + retry logic adds 5-10s but prevents low-quality output

### Contract-Driven Design is Working
- Zero hardcoded metric/hierarchy references in main agent code
- All configuration comes from `contract.yaml`
- Test fixtures appropriately use dataset-specific values
