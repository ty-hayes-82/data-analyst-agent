   - Already optimized with strict constraints:
     - 3-5 primary cards (optionally +2 supporting)
     - ≤28 words for `what_changed` and `why`
     - ≤35 words for narrative summary
   - JSON-only output (no markdown overhead)
   - Focused scope (single metric analysis)

2. **report_synthesis_agent** (`config/prompts/report_synthesis.md`):
   - Already optimized with strict constraints:
     - Narrative paragraphs ≤2 sentences (≤25 words each)

**Conclusion**: Slowness is inherent to LLM token generation, not prompt size. Both prompts are already tightly constrained. Further optimization would require:
- Model selection changes (faster/smaller models)
- Parallel execution (already implemented for some agents)
- Caching strategies (already implemented via `executive_brief_input_cache.json`)

No immediate action needed. Current performance is acceptable for production use.
